# 光栅图片 vs 矢量图
光栅图由二维的像素网格组成。每个像素存储了颜色和透明度的值。
主流的光栅图：JPEG（JPG），GIF，PNG
缺点：不能很好的支持缩放

矢量图由一些线条、形状和路径点组成。矢量图的各种信息并不是保存在像素点中，而是保存在精确的绘画指令中，而这些指令是完全独立于像素的。
主流的矢量图：SVG
优点：支持优秀的缩放

# 有损 vs 无损
有损：JPEG  体积小
无损：GIF，PNG  体积大

# JPEG：
支持数百万种颜色，比如展示摄影作品时，JPEG是个不错的选择。
所有设备都支持，兼容性好
不支持透明，如果打算用透明美化背景，不是明智的选择

# GIF
最多支持256种颜色的8位无损图片格式，体积小
支持透明
支持动画

# PNG
无损的8位和24位，多层级透明度的网络图片
支持透明
支持复杂的照片和图形
对于经常需要导出的图片，PNG的无损压缩保证没有质量退化

# SVG
XML的矢量文件类型
SVG 格式的图片通常要比光栅格式的图片小很多，特别是当你作了特别的优化并且启用 gzip 压缩的时候
它们是可缩放的，这意味着 SVG 图片能在各种分辨率的屏幕上展现的很清晰
你可以把 SVG 的代码丢到 HTML 中以减少请求数量
你可以用 CSS 来 DIY 任何 SVG
最酷的是，你可以用 JavaScript 或者 CSS 让你的 SVG 变成动画

# Webp
它以更小的体积糅合了 JPEG 和 PNG 的优点。
